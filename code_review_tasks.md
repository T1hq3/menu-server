# Пропозиції задач після ревʼю коду

1. **Друкарська помилка (UI-текст):**
   - Виправити назву розділу `Сети` на коректну форму `Сети/Сети?` згідно з затвердженою локалізацією меню (у коді є жорстко закодований список секцій, де можливий typo).
   - Файл: `main.py`, `SECTION_ORDER`.

2. **Функціональна помилка (валідація доступності PDF):**
   - У маршруті `/download` перевіряти не лише `STATUS["pdf_ready"]`, а і фактичне існування та ненульовий розмір `PDF_FILE`; зараз при рестарті процесу прапорець може лишатися `True`, але файл може бути відсутнім/пошкодженим.
   - Файл: `main.py`, функція `download_pdf()`.

3. **Покращення коментаря до коду:**
   - Замінити неінформативний коментар `# Дуже важливо` на пояснення *чому* додаються `authorization` header і cookie `token`, включно з форматом токена, який очікує API.
   - Файл: `main.py`, блок `login_and_get_session()`.

4. **Недотримання вимог документації/конфігурації:**
   - Додати `README.md` з обовʼязковими змінними середовища (`IDENTIFIER`, `PASSWORD`, `PORT`), кроками запуску та описом ендпоінтів (`/`, `/download`, `/status`); зараз ці вимоги імпліцитні в коді і не задокументовані.
   - Файли: новий `README.md` + посилання на `main.py`.

5. **Покращення тестування:**
   - Додати unit/integration тести для `build_html()` і Flask-роутів (`/status`, `/download`) з моками для файлової системи та стану `STATUS`, щоб перевіряти edge-caseʼи (відсутній PDF, порожні категорії, нульова ціна).
   - Файли: новий каталог `tests/`.
